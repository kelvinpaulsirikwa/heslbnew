## 1. Introduction

### 1.1 Purpose of this Manual
This manual provides comprehensive guidance on using the HESLB Website Management System. It covers both public website features and administrative functions.

### 1.2 Who Should Use This Manual
- **Website Visitors**: Students, parents, and general public seeking information about loans and scholarships
- **Content Administrators**: Staff responsible for managing website content
- **System Administrators**: IT staff managing users and system configurations

### 1.3 System Requirements
- **Browser**: Modern web browser (Chrome, Firefox, Edge, Safari)
- **Internet Connection**: Stable internet connection
- **Screen Resolution**: Minimum 1280x720 pixels recommended
- **For Admin Users**: Valid login credentials

## 2. System Overview

The HESLB Website is a comprehensive platform that provides:

1. **Email/Username**: Enter your registered email

#### 5.2.1 Force Password Change
- New users must change password on first login
- Follow the password change form
- Password requirements:
  - Minimum 8 characters

  - Include numbers

1. Click on your profile/user menu
2. Select "Logout"
3. Confirm logout

## 7. User Roles and Permissions

### 7.1 Understanding Roles

The system uses role-based access control with different permission levels:

#### 7.1.1 Super Administrator
- Full system access
- Can manage all content
- Can manage users and roles
- Can access all features

#### 7.1.2 Administrator
- Manage most content
- Cannot manage other administrators
- Limited user management

#### 7.1.3 Editor
- Create and edit content
- Cannot delete major items
- Cannot manage users

#### 7.1.4 Viewer
- Read-only access
- Can view admin panel
- Cannot make changes

### 7.2 Permission Restrictions

#### 7.2.1 Self-Management Restrictions
- Users cannot delete their own account
- Users cannot change their own role
- Users cannot block themselves

#### 7.2.2 Blocked User Restrictions
- Blocked users cannot log in
- Blocked users cannot perform any actions
- System prevents actions from blocked users

## 9. Troubleshooting

### 9.1 Login Issues

#### Problem: Cannot log in
**Solutions:**
1. Verify email and password are correct
2. Check Caps Lock is off
3. Clear browser cache and cookies
4. Try different browser
5. Contact system administrator if locked out

#### Problem: Forced to change password repeatedly
**Solutions:**
1. Ensure new password meets requirements
2. Use different password from previous ones
3. Check password confirmation matches
4. Contact administrator if persists

### 9.2 Content Issues

#### Problem: Cannot see uploaded content
**Solutions:**
1. Check if content status is "Active" or "Published"
2. Clear browser cache
3. Verify content is not scheduled for future date
4. Check if you have necessary permissions

#### Problem: Images not displaying
**Solutions:**
1. Verify image format (JPG, PNG recommended)
2. Check file size (maximum 100MB recommended)
3. Ensure image uploaded successfully
4. Check file permissions

### 9.3 File Upload Issues

#### Problem: Cannot upload file
**Solutions:**
1. Check file size (usually max 100MB for PDFs)
2. Verify file format is allowed
3. Check internet connection
4. Try compressing the file
5. Contact administrator if persists

#### Problem: PDF not opening
**Solutions:**
1. Verify PDF is not corrupted
2. Try re-uploading the file
3. Check PDF is not password-protected
4. Use PDF repair tool if necessary

### 9.4 Permission Issues

#### Problem: Cannot access certain features
**Solutions:**
1. Check your user role and permissions
2. Contact administrator to verify access level
3. Log out and log back in
4. Verify account is not blocked

#### Problem: Changes not saving
**Solutions:**
1. Check for validation errors on form
2. Ensure all required fields are filled
3. Check internet connection
4. Try submitting again
5. Contact administrator if data lost

### 9.5 General Issues

#### Problem: Page not loading
**Solutions:**
1. Refresh the page (F5)
2. Clear browser cache
3. Check internet connection
4. Try different browser
5. Contact IT support

#### Problem: Slow performance
**Solutions:**
1. Check internet speed
2. Close unnecessary browser tabs
3. Clear browser cache
4. Restart browser
5. Contact IT if system-wide

## 11. Best Practices

### 11.1 Content Management

#### Writing for the Web
1. **Keep it concise**: Use short paragraphs
2. **Use headings**: Break content into sections
3. **Bullet points**: Make lists scannable
4. **Active voice**: Write clearly and directly
5. **Proofread**: Check spelling and grammar

#### Image Guidelines
1. **Resolution**: Use high-quality images
2. **File size**: Optimize to under 500KB when possible
3. **Format**: Prefer JPG for photos, PNG for graphics
4. **Alt text**: Always add descriptive alt text
5. **Copyright**: Only use images you have rights to

#### Document Publishing
1. **File naming**: Use clear, descriptive names
2. **Accessibility**: Ensure PDFs are accessible
3. **Version control**: Include version/date in filename
4. **File size**: Compress large PDFs
5. **Metadata**: Add proper title and description

### 11.2 Security Best Practices

#### Password Security
1. Use strong, unique passwords
2. Change password regularly
3. Never share login credentials
4. Don't write passwords down
5. Use different passwords for different systems

#### Account Security
1. Log out when leaving computer
2. Don't leave admin panel open
3. Report suspicious activity
4. Verify before clicking links in emails
5. Keep browser updated

### 11.3 Workflow Recommendations

#### Content Approval Process
1. Draft content offline first
2. Review before uploading
3. Use draft status for review
4. Get necessary approvals
5. Schedule publication appropriately

#### Regular Maintenance
1. Review and update outdated content
2. Check for broken links monthly
3. Archive old news/events
4. Clean up unused files
5. Update FAQs regularly
